◆ makeConstructorFinisher()

function makeConstructorFinisher (   parser)
private

For method definitions that are constructors, create a function that will merge portions of the constructor's doclet into the class's doclet, provided that a doclet exists for the class.

Merging the constructor's documentation allows ES 2015 classes to be documented in a natural way, with separate JSDoc comments for the class and its constructor.

Parameters
{module:jsdoc/src/parser.Parser}parser - The JSDoc parser.
Returns
{function} A function that merges the constructor's doclet into the class's doclet.

◆ makeDefaultParamFinisher()

function makeDefaultParamFinisher ( )
private

For functions that may have at least one parameter with default values, create a function that will automatically add the parameters' default values to the function's documentation.

If any default value is already documented, the function's doclet will remain unchanged.

This function is only intended to handle default parameters whose node type is Syntax.Literal (string, numeric, and boolean literals). This is because more complex default values may include, for example, references to internal variables, which it may not make sense to include in documentation.

Returns
{function} A function that updates the function doclet to include the default values of parameters.
