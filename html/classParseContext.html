Parameters
stateThe current editor state.
fragmentsTree fragments that can be reused by incremental re-parses.
viewportThe current editor viewport (or some overapproximation thereof). Intended to be used for opportunistically avoiding work (in which case [`skipUntilInView`](https://codemirror.net/6/docs/ref/#language.ParseContext.skipUntilInView) should be called to make sure the parser is restarted when the skipped region becomes visible).
scheduleOnThis is where skipping parsers can register a promise that, when resolved, will schedule a new parse. It is cleared when the parse worker picks up the promise.

◆ getSkippingParser()

static ParseContext::getSkippingParser (   until)
static

Returns a parser intended to be used as placeholder when asynchronously loading a nested parser.

It'll skip its input and mark it as not-really-parsed, so that the next update will parse it again.

When until is given, a reparse will be scheduled when that promise resolves.
