Inside the {\ttfamily enter} function, calling {\ttfamily this.\+skip()} will prevent the node\textquotesingle{}s children being walked, or the {\ttfamily leave} function (which is optional) being called.

Call {\ttfamily this.\+replace(new\+\_\+node)} in either {\ttfamily enter} or {\ttfamily leave} to replace the current node with a new one.

Call {\ttfamily this.\+remove()} in either {\ttfamily enter} or {\ttfamily leave} to remove the current node.\hypertarget{md_node_modules_estree-walker_README_autotoc_md1130}{}\doxysection{Why not use estraverse?}\label{md_node_modules_estree-walker_README_autotoc_md1130}
The E\+S\+Tree spec is evolving to accommodate E\+S6/7. I\textquotesingle{}ve had a couple of experiences where \href{https://github.com/estools/estraverse}{\texttt{ estraverse}} was unable to handle an A\+ST generated by recent versions of acorn, because it hard-\/codes visitor keys.

estree-\/walker, by contrast, simply enumerates a node\textquotesingle{}s properties to find child nodes (and child lists of nodes), and is therefore resistant to spec changes. It\textquotesingle{}s also much smaller. (The performance, if you\textquotesingle{}re wondering, is basically identical.)
