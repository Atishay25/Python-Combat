Constructs a filter function which can be used to determine whether or not certain modules should be operated upon.

Parameters\+: {\ttfamily (include?\+: $<$picomatch$>$, exclude?\+: $<$picomatch$>$, options?\+: Object)}~\newline
 Returns\+: {\ttfamily String}\hypertarget{md_node_modules__0drollup_pluginutils_README_autotoc_md904}{}\doxyparagraph{$<$tt$>$include$<$/tt$>$ and $<$tt$>$exclude$<$/tt$>$}\label{md_node_modules__0drollup_pluginutils_README_autotoc_md904}
Type\+: {\ttfamily String $\vert$ Reg\+Exp $\vert$ Array\mbox{[}...String$\vert$\+Reg\+Exp\mbox{]}}~\newline


A valid \href{https://github.com/micromatch/picomatch\#globbing-features}{\texttt{ {\ttfamily picomatch}}} pattern, or array of patterns. If {\ttfamily options.\+include} is omitted or has zero length, filter will return {\ttfamily true} by default. Otherwise, an ID must match one or more of the {\ttfamily picomatch} patterns, and must not match any of the {\ttfamily options.\+exclude} patterns.

Note that {\ttfamily picomatch} patterns are very similar to \href{https://github.com/isaacs/minimatch\#readme}{\texttt{ {\ttfamily minimatch}}} patterns, and in most use cases, they are interchangeable. If you have more specific pattern matching needs, you can view \href{https://github.com/micromatch/picomatch\#library-comparisons}{\texttt{ this comparison table}} to learn more about where the libraries differ.\hypertarget{md_node_modules__0drollup_pluginutils_README_autotoc_md905}{}\doxyparagraph{$<$tt$>$options$<$/tt$>$}\label{md_node_modules__0drollup_pluginutils_README_autotoc_md905}

Type\+: {\ttfamily String $\vert$ Boolean $\vert$ null}

Optionally resolves the patterns against a directory other than {\ttfamily process.\+cwd()}. If a {\ttfamily String} is specified, then the value will be used as the base directory.
