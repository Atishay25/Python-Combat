\doxysubparagraph*{preset\+Name}

Markdown\+It provides named presets as a convenience to quickly enable/disable active syntax rules and options for common use cases.


\begin{DoxyItemize}
\item \href{https://github.com/markdown-it/markdown-it/blob/master/lib/presets/commonmark.js}{\texttt{ \char`\"{}commonmark\char`\"{}}} -\/ configures parser to strict \href{http://commonmark.org/}{\texttt{ Common\+Mark}} mode.
\item \href{https://github.com/markdown-it/markdown-it/blob/master/lib/presets/default.js}{\texttt{ default}} -\/ similar to G\+FM, used when no preset name given. Enables all available rules, but still without html, typographer \& autolinker.
\item \href{https://github.com/markdown-it/markdown-it/blob/master/lib/presets/zero.js}{\texttt{ \char`\"{}zero\char`\"{}}} -\/ all rules disabled. Useful to quickly setup your config via {\ttfamily .\mbox{\hyperlink{markdown-it_2lib_2index_8js_a44f74913494bbd4bd28f3ede02e96fe4}{enable()}}}. For example, when you need only {\ttfamily bold} and {\ttfamily italic} markup and nothing else.
\end{DoxyItemize}

\label{markdown-it_2lib_2index_8js_autotoc_md1325}%

\doxysubparagraph*{options\+:}


\begin{DoxyItemize}
\item {\bfseries{html}} -\/ {\ttfamily false}. Set {\ttfamily true} to enable H\+T\+ML tags in source. Be careful! That\textquotesingle{}s not safe! You may need external sanitizer to protect output from X\+SS. It\textquotesingle{}s better to extend features via plugins, instead of enabling H\+T\+ML.
\item {\bfseries{xhtml\+Out}} -\/ {\ttfamily false}. Set {\ttfamily true} to add \textquotesingle{}/\textquotesingle{} when closing single tags ({\ttfamily $<$br /$>$}). This is needed only for full Common\+Mark compatibility. In real world you will need H\+T\+ML output.
\item {\bfseries{breaks}} -\/ {\ttfamily false}. Set {\ttfamily true} to convert {\ttfamily \textbackslash{}n} in paragraphs into {\ttfamily $<$br$>$}.
\item {\bfseries{lang\+Prefix}} -\/ {\ttfamily language-\/}. C\+SS language class prefix for fenced blocks. Can be useful for external highlighters.
\item {\bfseries{linkify}} -\/ {\ttfamily false}. Set {\ttfamily true} to autoconvert U\+R\+L-\/like text to links.
\item {\bfseries{typographer}} -\/ {\ttfamily false}. Set {\ttfamily true} to enable \href{https://github.com/markdown-it/markdown-it/blob/master/lib/rules_core/replacements.js}{\texttt{ some language-\/neutral replacement}} + quotes beautification (smartquotes).
\item {\bfseries{quotes}} -\/ {\ttfamily “”‘’}, String or Array. Double + single quotes replacement pairs, when typographer enabled and smartquotes on. For example, you can use `\textquotesingle{}«»„“'{\ttfamily for Russian,}\textquotesingle{}„“‚‘\textquotesingle{}{\ttfamily for German, and }\mbox{[}\textquotesingle{}«\textbackslash{}x\+A0\textquotesingle{}, \textquotesingle{}\textbackslash{}x\+A0»\textquotesingle{}, \textquotesingle{}‹\textbackslash{}x\+A0\textquotesingle{}, \textquotesingle{}\textbackslash{}x\+A0›\textquotesingle{}\mbox{]}{\ttfamily for French (including nbsp).}
\item {\ttfamily \+\_\+\+\_\+highlight\+\_\+\+\_\+ -\/}null{\ttfamily . Highlighter function for fenced code blocks. Highlighter}function (str, lang)\`{} should return escaped H\+T\+ML. It can also return empty string if the source was not changed and should be escaped externaly. If result starts with $<$pre... internal wrapper is skipped.
\end{DoxyItemize}
