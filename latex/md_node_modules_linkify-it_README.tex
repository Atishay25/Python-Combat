{\bfseries{\href{http://markdown-it.github.io/linkify-it/doc}{\texttt{ A\+PI documentation}}}}\hypertarget{md_node_modules_linkify-it_README_autotoc_md1301}{}\doxysubsection{new Linkify\+It(schemas, options)}\label{md_node_modules_linkify-it_README_autotoc_md1301}
Creates new linkifier instance with optional additional schemas. Can be called without {\ttfamily new} keyword for convenience.

By default understands\+:


\begin{DoxyItemize}
\item {\ttfamily http(s)\+://...} , {\ttfamily \href{ftp://..}{\texttt{ ftp\+://..}}.}, {\ttfamily mailto\+:...} \& {\ttfamily //...} links
\item \char`\"{}fuzzy\char`\"{} links and emails (google.\+com, \href{mailto:foo@bar.com}{\texttt{ foo@bar.\+com}}).
\end{DoxyItemize}

{\ttfamily schemas} is an object, where each key/value describes protocol/rule\+:


\begin{DoxyItemize}
\item {\bfseries{key}} -\/ link prefix (usually, protocol name with {\ttfamily \+:} at the end, {\ttfamily skype\+:} for example). {\ttfamily linkify-\/it} makes sure that prefix is not preceded with alphanumeric char.
\item {\bfseries{value}} -\/ rule to check tail after link prefix
\begin{DoxyItemize}
\item {\itshape String} -\/ just alias to existing rule
\item {\itshape Object}
\begin{DoxyItemize}
\item {\itshape validate} -\/ either a {\ttfamily Reg\+Exp} (start with {\ttfamily $^\wedge$}, and don\textquotesingle{}t include the link prefix itself), or a validator function which, given arguments {\itshape text}, {\itshape pos}, and {\itshape self}, returns the length of a match in {\itshape text} starting at index {\itshape pos}. {\itshape pos} is the index right after the link prefix. {\itshape self} can be used to access the linkify object to cache data.
\item {\itshape normalize} -\/ optional function to normalize text \& url of matched result (for example, for twitter mentions).
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}

{\ttfamily options}\+:


\begin{DoxyItemize}
\item {\bfseries{fuzzy\+Link}} -\/ recognize U\+R\+L-\/s without {\ttfamily http(s)\+://} head. Default {\ttfamily true}.
\item {\bfseries{fuzzy\+IP}} -\/ allow I\+Ps in fuzzy links above. Can conflict with some texts like version numbers. Default {\ttfamily false}.
\item {\bfseries{fuzzy\+Email}} -\/ recognize emails without {\ttfamily mailto\+:} prefix. Default {\ttfamily true}.
\item {\bfseries{---}} -\/ set {\ttfamily true} to terminate link with {\ttfamily -\/-\/-\/} (if it\textquotesingle{}s considered as long dash).
\end{DoxyItemize}\hypertarget{md_node_modules_linkify-it_README_autotoc_md1302}{}\doxysubsection{.\+test(text)}\label{md_node_modules_linkify-it_README_autotoc_md1302}

\item {\bfseries{text}} -\/ normalized text
\item {\bfseries{url}} -\/ link, generated from matched text
