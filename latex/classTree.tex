\doxysubsection{Detailed Description}
A piece of syntax tree. 

There are two ways to approach these trees\+: the way they are actually stored in memory, and the convenient way.

Syntax trees are stored as a tree of {\ttfamily \mbox{\hyperlink{classTree}{Tree}}} and {\ttfamily \mbox{\hyperlink{classTreeBuffer}{Tree\+Buffer}}} objects. By packing detail information into {\ttfamily \mbox{\hyperlink{classTreeBuffer}{Tree\+Buffer}}} leaf nodes, the representation is made a lot more memory-\/efficient.

However, when you want to actually work with tree nodes, this representation is very awkward, so most client code will want to use the \href{\#common.TreeCursor}{\texttt{ {\ttfamily Tree\+Cursor}}} or \href{\#common.SyntaxNode}{\texttt{ {\ttfamily Syntax\+Node}}} interface instead, which provides a view on some part of this data structure, and can be used to move around to adjacent nodes.
