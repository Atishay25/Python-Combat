J\+S\+Doc supports stable versions of Node.\+js 12.\+0.\+0 and later. You can install J\+S\+Doc globally or in your project\textquotesingle{}s {\ttfamily node\+\_\+modules} folder.

To install the latest version on npm globally (might require {\ttfamily sudo}; \href{https://docs.npmjs.com/resolving-eacces-permissions-errors-when-installing-packages-globally}{\texttt{ learn how to fix this}})\+: \begin{DoxyVerb}npm install -g jsdoc
\end{DoxyVerb}


To install the latest version on npm locally and save it in your package\textquotesingle{}s {\ttfamily package.\+json} file\+: \begin{DoxyVerb}npm install --save-dev jsdoc
\end{DoxyVerb}


To install the latest development version locally, without updating your project\textquotesingle{}s {\ttfamily package.\+json} file\+: \begin{DoxyVerb}npm install git+https://github.com/jsdoc/jsdoc.git
\end{DoxyVerb}


If you installed J\+S\+Doc locally, the J\+S\+Doc command-\/line tool is available in {\ttfamily ./node\+\_\+modules/.bin}. To generate documentation for the file {\ttfamily your\+Java\+Script\+File.\+js}\+: \begin{DoxyVerb}./node_modules/.bin/jsdoc yourJavaScriptFile.js
\end{DoxyVerb}


If you installed J\+S\+Doc globally, run the {\ttfamily jsdoc} command\+: \begin{DoxyVerb}jsdoc yourJavaScriptFile.js
\end{DoxyVerb}


By default, the generated documentation is saved in a directory named {\ttfamily out}. You can use the {\ttfamily -\/-\/destination} ({\ttfamily -\/d}) option to specify another directory.
