A Java\+Script parser for \href{https://github.com/google/closure-compiler/wiki/Annotating-JavaScript-for-the-Closure-Compiler\#type-expressions}{\texttt{ Google Closure Compiler}} and \href{https://github.com/jsdoc/jsdoc}{\texttt{ J\+S\+Doc}} type expressions.

\mbox{\hyperlink{classCatharsis}{Catharsis}} is designed to be\+:


\begin{DoxyItemize}
\item {\bfseries{Accurate}}. \mbox{\hyperlink{classCatharsis}{Catharsis}} is based on a \href{https://pegjs.org/}{\texttt{ P\+E\+G.\+js}} grammar that\textquotesingle{}s designed to handle any valid type expression. It uses a thorough test suite to verify the parser\textquotesingle{}s accuracy.
\item {\bfseries{Fast}}. \mbox{\hyperlink{classParse}{Parse}} results are cached, so the parser is invoked only when necessary.
\item {\bfseries{Flexible}}. \mbox{\hyperlink{classCatharsis}{Catharsis}} can convert a parse result back into a type expression, or into a description of the type expression. In addition, \mbox{\hyperlink{classCatharsis}{Catharsis}} can parse \href{https://github.com/jsdoc/jsdoc}{\texttt{ J\+S\+Doc}}-\/style type expressions.

\mbox{\hyperlink{classParse}{Parse}} a type expression, and return the parse results. Throws an error if the type expression cannot be parsed.

When called without options, \mbox{\hyperlink{classCatharsis}{Catharsis}} attempts to parse type expressions in the same way as Closure Compiler. When the {\ttfamily jsdoc} option is enabled, \mbox{\hyperlink{classCatharsis}{Catharsis}} can also parse several kinds of type expressions that are permitted in \href{https://github.com/jsdoc/jsdoc}{\texttt{ J\+S\+Doc}}\+:


\begin{DoxyItemize}
\item The string {\ttfamily function} is treated as a function type with no parameters.
\item You can omit the period from type applications. For example, {\ttfamily Array.$<$string$>$} and {\ttfamily Array$<$string$>$} are parsed in the same way.
\item If can append {\ttfamily \mbox{[}\mbox{]}} to a name expression (for example, {\ttfamily string\mbox{[}\mbox{]}}), it is interpreted as a type application with the expression {\ttfamily Array} (for example, {\ttfamily Array.$<$string$>$}).
\item Name expressions can contain the characters {\ttfamily \#}, {\ttfamily $\sim$}, {\ttfamily \+:}, and {\ttfamily /}.
\item Name expressions can contain a suffix that is similar to a function signature (for example, {\ttfamily My\+Class(foo, bar)}).
\item Name expressions can contain a reserved word.
\item Record types can use types other than name expressions for keys.
\end{DoxyItemize}\hypertarget{md_node_modules_catharsis_README_autotoc_md1027}{}\doxysubsubsection{Parameters}\label{md_node_modules_catharsis_README_autotoc_md1027}

Convert a parsed type to a description of the type expression. This method is especially useful if your users are not familiar with the syntax for type expressions.

The {\ttfamily \mbox{\hyperlink{catharsis_8js_a3a96e405b0f405dfaea25b99c006e437}{describe()}}} method returns the description in two formats\+:


\begin{DoxyItemize}
\item {\bfseries{Simple format}}. A string that provides a complete description of the type expression.
\item {\bfseries{Extended format}}. An object that separates out some of the details about the outermost type expression, such as whether the type is optional, nullable, or repeatable.
\end{DoxyItemize}

\item 0.\+4.\+0 (March 2013)\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classCatharsis}{Catharsis}} now supports a lenient parsing option that can parse several kinds of malformed type expressions. See the documentation for details.
\item The objects containing parse results are now frozen.
\item The objects containing parse results now have two non-\/enumerable properties\+:
\begin{DoxyItemize}
\item {\ttfamily lenient}\+: A boolean indicating whether the type expression was parsed in lenient mode.
\item {\ttfamily type\+Expression}\+: A string containing the original type expression.
\end{DoxyItemize}
\item The {\ttfamily \mbox{\hyperlink{catharsis_8js_a4165071c679fe5c29c3c3f92375bfabd}{stringify()}}} method now honors the {\ttfamily use\+Cache} option. If a parsed type includes a {\ttfamily type\+Expression} property, and {\ttfamily use\+Cache} is not set to {\ttfamily false}, the stringified type will be identical to the original type expression.
\end{DoxyItemize}
\item 0.\+3.\+1 (March 2013)\+: Type expressions that begin with a reserved word, such as {\ttfamily integer}, are now parsed correctly.
\item 0.\+3.\+0 (March 2013)\+:
\begin{DoxyItemize}
\item The {\ttfamily \mbox{\hyperlink{_0dbabel_2parser_2lib_2index_8js_a6fd05206266746979959a2bac1f80a2b}{parse()}}} and {\ttfamily \mbox{\hyperlink{catharsis_8js_a4165071c679fe5c29c3c3f92375bfabd}{stringify()}}} methods are now synchronous, and the {\ttfamily parse\+Sync()} and {\ttfamily stringify\+Sync()} methods have been removed. {\bfseries{Note}}\+: This change is not backwards-\/compatible with previous versions.
\item The parse results now use a significantly different format from previous versions. The new format is more expressive and is similar, but not identical, to the format used by the \href{https://github.com/eslint/doctrine}{\texttt{ doctrine}} parser. {\bfseries{Note}}\+: This change is not backwards-\/compatible with previous versions.
\item Name expressions that contain a reserved word now include a {\ttfamily reserved\+Word\+: true} property.
\item Union types that are optional or nullable, or that can be repeated, are now parsed and stringified correctly.
\item Optional function types and record types are now parsed and stringified correctly.
\item Function types now longer include {\ttfamily new} or {\ttfamily this} properties unless the properties are defined in the type expression. In addition, the {\ttfamily new} and {\ttfamily this} properties can now use any type expression.
\item In record types, the key for a field type can now use any type expression.
\item Standalone single-\/character literals, such as A\+LL ({\ttfamily $\ast$}), are now parsed and stringified correctly.
\item {\ttfamily null} and {\ttfamily undefined} literals with additional properties, such as {\ttfamily repeatable}, are now stringified correctly.
\end{DoxyItemize}
