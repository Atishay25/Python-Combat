xmlcreate is a Node.\+js module that can be used to build X\+ML using a simple A\+PI.\hypertarget{md_node_modules_xmlcreate_README_autotoc_md1704}{}\doxysection{Features}\label{md_node_modules_xmlcreate_README_autotoc_md1704}
xmlcreate allows you to use a series of chained function calls to build an X\+ML tree.

Once the tree is built, it can be serialized to text. The formatting of the text is customizable.

xmlcreate can perform some basic validation to check that the resulting X\+ML is well-\/formed.\hypertarget{md_node_modules_xmlcreate_README_autotoc_md1705}{}\doxysection{Installation}\label{md_node_modules_xmlcreate_README_autotoc_md1705}
The easiest way to install xmlcreate is using npm\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm install xmlcreate}
\end{DoxyCode}


You can also build xmlcreate from source using npm\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git clone https://github.com/michaelkourlas/node-\/xmlcreate.git}
\DoxyCodeLine{npm install}
\DoxyCodeLine{npm run-\/script build}
\end{DoxyCode}


The {\ttfamily build} script will build the production variant of xmlcreate, run all tests, and build the documentation.

You can build the production variant without running tests using the script {\ttfamily prod}. You can also build the development version using the script {\ttfamily dev}. The only difference between the two is that the development version includes source maps.\hypertarget{md_node_modules_xmlcreate_README_autotoc_md1706}{}\doxysection{Usage}\label{md_node_modules_xmlcreate_README_autotoc_md1706}
The documentation for the current version is available \href{http://www.kourlas.com/node-xmlcreate/docs/2.0.4/}{\texttt{ here}}.

You can also build the documentation using npm\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm run-\/script docs}
\end{DoxyCode}
