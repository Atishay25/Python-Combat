Update the list of doclets to be added to another symbol. 

We add only one doclet per longname. For example\+: If {\ttfamily ClassA} inherits from two classes that both use the same method name, {\ttfamily ClassA} gets docs for one method rather than two.

Also, the last symbol wins for any given longname. For example\+: If you write {\ttfamily @extends Class1 @extends Class2}, and both classes have an instance method called {\ttfamily my\+Method}, you get the docs from {\ttfamily Class2\#my\+Method}.

Add and update doclets to reflect implementations of interfaces. 

For example, if {\ttfamily InterfaceA} has the instance method {\ttfamily my\+Method}, and {\ttfamily ClassA} implements {\ttfamily InterfaceA}, calling this method does the following\+:


\begin{DoxyItemize}
\item Updates {\ttfamily InterfaceA} to indicate that it is implemented by {\ttfamily ClassA}
\item Updates {\ttfamily InterfaceA\#my\+Method} to indicate that it is implemented by {\ttfamily ClassA\#my\+Method}
\item Updates {\ttfamily ClassA\#my\+Method} to indicate that it implements {\ttfamily InterfaceA\#my\+Method}
\end{DoxyItemize}

If {\ttfamily ClassA\#my\+Method} used the {\ttfamily @override} or {\ttfamily @inheritdoc} tag, calling this method would also generate a new doclet that reflects the interface\textquotesingle{}s documentation for {\ttfamily InterfaceA\#my\+Method}.
