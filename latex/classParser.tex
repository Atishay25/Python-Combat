{\footnotesize\ttfamily Parser\+::resolve\+Property\+Parents (\begin{DoxyParamCaption}\item[{}]{\{parent\} }\end{DoxyParamCaption})}



Given an A\+ST node representing an object property, find the doclets for the parent object or objects. 

If the object is part of a simple assignment (for example, {\ttfamily var foo = \{ x\+: 1 \}}), this method returns a single doclet (in this case, the doclet for {\ttfamily foo}).

If the object is part of a chained assignment (for example, {\ttfamily var foo = exports.\+F\+OO = \{ x\+: 1 \}}, this method returns multiple doclets (in this case, the doclets for {\ttfamily foo} and {\ttfamily exports.\+F\+OO}).

{\footnotesize\ttfamily Parser\+::start\+Parse (\begin{DoxyParamCaption}\item[{}]{input,  }\item[{}]{fragments,  }\item[{}]{ranges }\end{DoxyParamCaption})}



Start a parse, returning a \href{\#common.PartialParse}{\texttt{ partial parse}} object. 

\href{\#common.TreeFragment}{\texttt{ {\ttfamily fragments}}} can be passed in to make the parse incremental.

By default, the entire input is parsed. You can pass {\ttfamily ranges}, which should be a sorted array of non-\/empty, non-\/overlapping ranges, to parse only those ranges. The tree returned in that case will start at {\ttfamily ranges\mbox{[}0\mbox{]}.from}.
