\doxysubsubsection{\texorpdfstring{functionDoclets}{functionDoclets}}
{\footnotesize\ttfamily let function\+Doclets}



The Overload Helper plugin automatically adds a signature-\/like string to the longnames of overloaded functions and methods. 

In J\+S\+Doc, this string is known as a {\itshape variation}. (The longnames of overloaded constructor functions are {\itshape not} updated, so that J\+S\+Doc can identify the class\textquotesingle{} members correctly.)

Using this plugin allows you to link to overloaded functions without manually adding {\ttfamily @variation} tags to your documentation.

For example, suppose your code includes a function named {\ttfamily foo} that you can call in the following ways\+:


\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{plugins_2test_2fixtures_2markdown_8js_af5f326655c1ca950dfb28952255e75c9}{foo()}}}
\item {\ttfamily foo(bar)}
\item {\ttfamily foo(bar, baz)} (where {\ttfamily baz} is repeatable)
\end{DoxyItemize}

This plugin assigns the following variations and longnames to each version of {\ttfamily foo}\+:


\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{plugins_2test_2fixtures_2markdown_8js_af5f326655c1ca950dfb28952255e75c9}{foo()}}} gets the variation {\ttfamily ()} and the longname {\ttfamily \mbox{\hyperlink{plugins_2test_2fixtures_2markdown_8js_af5f326655c1ca950dfb28952255e75c9}{foo()}}}.
\item {\ttfamily foo(bar)} gets the variation {\ttfamily (bar)} and the longname {\ttfamily foo(bar)}.
\item {\ttfamily foo(bar, baz)} (where {\ttfamily baz} is repeatable) gets the variation {\ttfamily (bar, ...baz)} and the longname {\ttfamily foo(bar, ...\+baz)}.
\end{DoxyItemize}

You can then link to these functions with {\ttfamily \{@link \mbox{\hyperlink{plugins_2test_2fixtures_2markdown_8js_af5f326655c1ca950dfb28952255e75c9}{foo()}}\}}, {\ttfamily \{@link foo(bar)\}}, and {\ttfamily \{@link foo(bar, ...\+baz)}. Note that the variation is based on the names of the function parameters, {\itshape not} their types.

If you prefer to manually assign variations to certain functions, you can still do so with the {\ttfamily @variation} tag. This plugin will not change these variations or add more variations for that function, as long as the variations you\textquotesingle{}ve defined result in unique longnames.

If an overloaded function includes multiple signatures with the same parameter names, the plugin will assign numeric variations instead, starting at {\ttfamily (1)} and counting upwards.
